# Ciber-DIO

Neste trabalho, buscamos compreender como funcionam os ataques de força bruta em diferentes serviços, como FTP, aplicações web e SMB. Esses ataques consistem basicamente em tentar diversas combinações de usuário e senha até encontrar um acesso válido, explorando principalmente credenciais fracas ou mal configuradas.

Para colocar isso em prática, utilizamos o Kali Linux, uma distribuição voltada para testes de segurança, junto com a ferramenta Medusa, que permite automatizar tentativas de autenticação em vários serviços. Todo o processo foi realizado em um ambiente controlado, garantindo que os testes fossem feitos de forma ética e segura.

Além da execução dos testes, também nos preocupamos em documentar cada etapa de forma clara e estruturada, registrando comandos utilizados, resultados obtidos e possíveis dificuldades encontradas. Isso é essencial para garantir reprodutibilidade e organização em auditorias de segurança.

Por fim, conseguimos reconhecer vulnerabilidades comuns, como senhas fracas, ausência de bloqueio por tentativas de login e serviços expostos. A partir disso, propomos medidas de mitigação, como o uso de senhas fortes, autenticação multifator e limitação de tentativas de acesso, aumentando significativamente a segurança dos sistemas.
